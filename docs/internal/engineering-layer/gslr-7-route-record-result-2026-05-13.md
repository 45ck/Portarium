# GSLR-7 Route-Record Result: 2026-05-13

Status: R&D boundary, no runtime ingestion  
Tracking bead: `bead-1247`  
Companion prompt-language bead: `prompt-language-gslr22`

## Decision

Portarium should keep route-decision record generation on `frontier-baseline`
for now.

GSLR-6 remains useful: the exact scaffolded sanitizer shape passed three local
repeats. GSLR-7 shows that this does not yet generalize to a route-record
builder where the local model still owns policy tables and output envelopes.

- live local attempts.

The deterministic fake-live proof passed.

The live local attempts failed:

| Attempt | Result | Failure boundary                         |
| ------- | ------ | ---------------------------------------- |
| v1      | fail   | `selectedRoute.arm` was missing          |
| v2      | fail   | `oracle command` unsafe key was accepted |

## Portarium Meaning

This is a strong product boundary result.

The next Portarium engineering-layer primitive should not ingest prompt-language
manifests directly. It should first define deterministic scaffold ownership:

- normalized policy tables are generated by PL or Portarium code;
- required output envelopes are generated by PL or Portarium code;
- local models fill only small predicates;
- Codex/frontier diagnoses the first failure or owns the route.

## Still Blocked

Do not create:

- live GSLR manifest ingestion;
- live Cockpit engineering cards from prompt-language runs;
- queues or database tables for route records;
- runtime policy decisions based on prompt-language manifests;
- MC connector observation or school-data movement.
